## Sprint 1 – Project Overview

### Description

In Sprint 1, we began development of a **console-based Trading Card Game Tournament Booking Platform**.

The goal of the platform is to support players and organizers within the trading card game community (e.g., Magic: The Gathering, Pokémon TCG, Yu-Gi-Oh!, Flesh & Blood) by providing a system to create, discover, and manage tournaments.

---

### Core Functionality

The platform supports two main user roles:

- **Organizers**
    - Create and manage tournaments
    - Define tournament details such as format, date & time, venue, casual or prize tournament, entry fee, and player capacity

- **Players**
    - Register accounts and log in
    - Browse available tournaments
    - View tournament details

The system also ensures that tournament data is structured and easy to access.

---

### Features Implemented in Sprint 1

- User account registration
- User login system
- Basic user profile interface
- Creating tournaments
- Viewing and listing tournaments

---

### Technical Focus

During this sprint, we focused on establishing the **core structure of the application**, including:

- Setting up the project architecture
- Implementing core classes (e.g., User, Tournament)
- Building a functional console menu system
- Ensuring clean separation between data models and UI

---

### Outcome

By the end of Sprint 1, we had a working foundation of the application.  
Users could register, log in, and create/view tournaments through a console interface.
