## Sprint 2 – Development Progress

### Overview

Sprint 2 focused on expanding the platform with more advanced functionality, particularly around bookings, validation, and user interaction with tournaments.

---

### Features Implemented in Sprint 2

- Booking / registration system for tournaments
- Capacity management (preventing overbooking)
- Basic input validation (e.g., valid data, constraints)
- Tournament review and rating system

---

### Technical Focus

During this sprint, we focused on:

- Implementing relationships between users, tournaments, and bookings
- Ensuring data integrity through validation and constraints
- Using C# features such as collections and LINQ for managing and querying data
- Improving overall application flow and user interaction

---

### Challenges

Towards the end of Sprint 2, we experienced issues with GitHub being temporarily unavailable.  
As a result:

- Sprint 3 tasks were planned and tracked manually
- Tasks were later added to the GitHub project board once service was restored

This caused a minor disruption in workflow but did not significantly impact development progress.

---

### Outcome

By the end of Sprint 2, the application supported full tournament participation flow:

- Users could browse tournaments
- Register for events
- Leave reviews after participation

This marked a major step toward completing the core functionality of the platform.
